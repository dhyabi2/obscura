The core idea

Your spend hides in the whole crowd

Drag the slider to grow the network. In Obscura, every output you see is an equally plausible spender — adoption itself is the privacy.

No decoys. No selection. No leakage.

Ring-signature coins pick a handful of decoys with a heuristic — and that heuristic leaks. Obscura proves membership in the accumulator of all outputs, so there is nothing to select and nothing to fingerprint.

Because the anonymity set is the UTXO set, it grows for free with every transaction the network ever makes.

Anatomy of a private transaction

Four layers of privacy, one transaction

Obscura layers four independent techniques so senders, recipients, amounts, and the act of spending are all hidden.

Stealth addresses

Your public address never appears on chain. The sender derives a fresh, one-time output key for every payment using Diffie–Hellman, so no two payments to you are linkable — even by you, without your view key.

P = H(r·A)·G + B  ·  R = r·G

A 1-byte view tag lets your wallet skip ~255/256 of non-yours outputs while scanning.

Confidential amounts

Values are sealed inside Pedersen commitments. Bit-decomposition range proofs guarantee no amount is negative and nothing is created from nothing, while a conservation proof shows inputs equal outputs — all without revealing a single number.

C = a·G + v·H  ·  Σ in = Σ out + fee

Global-set sender anonymity

A transparent zk-STARK membership proof hides which output is being spent, with a recipient-secret nullifier that prevents double-spends and stops even the sender from linking the coin. Proofs are logarithmic in set size, so the set can be huge.

Combined with the epoch-sharded commitment tree and class-group accumulator, the effective anonymity set is the whole chain, not a ring.

No trusted setup

The accumulator runs over a group of unknown order, an imaginary-quadratic class group needing no setup, or an RSA-2048 nothing-up-my-sleeve modulus. Spends are authenticated by ownership and value-equality proofs against the UTXO set, closing theft, inflation, and double-spend independently of the anonymity machinery. The transparent zk-STARK and class-group accumulator need no trusted setup. The live default binds ownership with a recipient-secret nullifier, so a sender can neither steal nor link the spend.

Optional Tor transport and Dandelion++ hide transaction origin at the network layer.

Tokenomics

Mine, hold, and grow the network

A smooth, front-loaded emission with a perpetual tail — security funded forever, no halving cliffs.

⛏️

Mine

Memory-hard VM proof-of-work: a memory-hard cache, a per-nonce scratchpad, and a randomized int/float VM. 120-second blocks, LWMA difficulty.

Time-lock outputs to earn from a bounded incentive pool (5% of emission) — provably, without revealing your amount.

A capped, decaying referral bonus that never mints new supply — sharing is a fee-share, not inflation.

Get started

From zero to a private transfer

Pure Go, CGO-free, single static binaries for Linux, macOS, and Windows. No runtime dependencies.

quickstart.sh
# build node, wallet, and the standalone miner
go build -o bin/obscura-node   ./cmd/obscura-node
go build -o bin/obscura-wallet ./cmd/obscura-wallet
go build -o bin/obscura-miner  ./cmd/obscura-miner

# create an encrypted wallet (prints a Base58 address)
./bin/obscura-wallet new --passphrase "your secret"

# start a node that mines to your wallet
./bin/obscura-node --mine --mine-address <your-address>

# check balance and send a confidential payment
./bin/obscura-wallet balance
./bin/obscura-wallet send --to <address> --amount 5 --fee auto
