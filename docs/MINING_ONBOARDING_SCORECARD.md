# How Easy Is It to Join Obscura and Mine? — Scorecard

**Verdict:** Mining Obscura is trivially easy to *start solo on your own fresh chain* (one binary, one flag, auto-wallet), but genuinely hard to *join the live network and earn steadily* — there are no baked-in seeds, no NAT traversal, no mining pool, and a proof-of-retrievability rule that forces self-hosted miners to run a full, synced node.

**Overall Ease Score: 71/100**

## Full Factor Table

## Top 3 Strengths

1. **Cross-platform single binary (95/100)** — verified pure-Go, CGO-free, cross-compiles to all major OS/arch with real per-platform archives already in dist/.
2. **Built-in miner (92/100)** — mining is a single in-process flag with an auto-created persistent payout wallet; no third-party mining software ever needed.
3. **Zero/minimal external dependencies (90/100)** — a static binary with no DB, no shared crypto libs, no Tor and no third-party RPC required by default; building needs only Go 1.24+ and 4 pure-Go modules.

## Top 5 Friction Points

1. **NAT traversal / inbound-port (35/100)** — no UPnP/NAT-PMP/PCP; nodes behind home NAT are inbound-unreachable and must hand-forward TCP 18080 with no in-app help or docs.
2. **Solo and pool mining support (40/100)** — zero stratum/pool/share code; small miners get all-or-nothing solo variance with no way to smooth payouts.
3. **Tor / privacy connectivity (40/100)** — works but expert-gated: manual Tor daemon + hand-built hidden service, no auto-provisioning, no bundled Tor.
4. **Seedless self-discovery (45/100)** — first join is not seedless; empty default seeds, no fallback, no DNS seeder, so a newcomer must paste a live host:port out-of-band.
5. **Proof-of-Retrievability full-node barrier (62/100)** — a self-hosted miner must sync and retain ~2 weeks of block bodies before `--mine` will succeed; no pool to offload to.

## Fastest Path to Easier Onboarding

The four lowest scores share two root causes — *you can't find the network*, and *you must run a heavyweight full node alone* — so the highest-leverage fixes are:

1. **Bake in a `--network testnet` preset (seeds + consensus params).** This single change directly lifts the two cheapest big wins — Seedless discovery (45) and the manual-`--seeds` caveat dragging on One-command start (88) and Config (80) — and removes the desync footgun, turning `obscura-node --mine` from "solo chain" into "joins the live network."
2. **Add a Stratum-style pool / template proxy with share-based coinbase splitting.** This is the only change that simultaneously raises Pool support (40), the PoR full-node barrier (62), Time-to-first-block (78), and Reward clarity (72), because it lets small miners earn steady, low-variance income without each running a full PoR-capable node — the single biggest structural barrier.
3. **Add UPnP + NAT-PMP/PCP auto port-mapping with a reachability self-test.** The lowest score (35) and the cheapest to move; it converts a silent inbound-unreachable failure into automatic inbound connectivity for typical home users.
4. **Publish prebuilt, checksummed binaries and a `curl | tar` install.** Removes the Go-toolchain prerequisite that quietly caps Built-in miner (92), Single binary (95), Dependencies (90), and Docs (68) for true non-experts.
5. **Fix the docs/code drift** (README "Argon2id" vs the shipped RandomX-style backend; undocumented PoR full-node rule) — near-zero effort, and it removes the trust erosion that holds Documentation (68) down.

Items 1–3 attack the lowest-scoring, highest-friction factors directly; items 4–5 are cheap, broad multipliers. Landing all five would plausibly move the overall from 71 into the low-to-mid 80s.
