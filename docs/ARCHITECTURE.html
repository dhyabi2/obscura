1 · Network & P2P

how nodes find each other & stay uncensorable

Self-discovering gossip mesh CENSORSHIP-RESISTANT

Nodes auto-discover peers — no hardcoded seeds, no DNS-seed single point of failure. Bootstrap nodes are pure convenience; once a node has a few peers it keeps the full mesh, so seizing the original servers cannot stop the network.

  • PEX — peers exchange address lists (getaddr/addr)
  • Self-address discovery — a node learns its own public IP from ≥2 peers' observations (Bitcoin "addr-me"); no config
  • Persistent address book — survives restart, evicts dead peers by failure count
  • Routable-only PEX — never advertises undialable (0.0.0.0) addresses

Transport privacy & reachability

Metadata-resistant transport so even the act of transacting is private, and NAT'd home users stay reachable.

  • Tor support — route P2P over Tor, advertise a .onion (also solves NAT)
  • Dandelion++ — tx stem/fluff relay hides the originating node
  • Onion-only mode — fail-closed: never mixes clearnet peers into the Tor set

DoS hardening

The gossip layer is built to absorb hostile peers without crashing or being flooded.

  • Magic + version handshake — rejects cross-network / wrong-fork peers
  • Ban scoring + per-IP caps — misbehavior is penalized and evicted
  • Backoff-with-jitter dialing, read/write deadlines, peer caps
  • Panic isolation — a bad message drops one peer, never the node

Mempool & ingestion

Admission is parallel and the pool is bounded, so a transaction flood degrades gracefully instead of OOM-ing.

  • Bounded mempool — capped txs/bytes with fee-priority eviction
  • Parallel proof verification — concurrent admission across cores; verified-proof cache avoids re-checking at block time
  • Min fee-per-byte — anti-spam floor

2 · Consensus & Proof-of-Work

how blocks are made, ordered & kept honest

RandomX-style PoW

Memory-hard, CPU-friendly proof-of-work that resists ASIC/GPU centralization — anyone with a CPU can mine.

  • VM-based hashing — memory-hard, egalitarian
  • Per-epoch seed rotation — seed taken from confirmed history so it can't be ground
  • LWMA difficulty retarget — smooth, manipulation-resistant
  • Miner pacing — production capped to the target interval (no block-spam starving P2P/RPC)

Proof-of-Retrievability MINERS = FULL NODES

Consensus forces block producers to hold full history: to mine, you must answer pseudo-random "prove you still have transaction X in block Y" challenges. A pruned node literally cannot build a valid block.

  • Challenge — k random (block,leaf) pairs from the parent hash; unforgeable, ungrindable
  • Proof — challenged tx bytes + merkle branch vs the committed header root
  • Asymmetric — only the miner needs bodies; pruned nodes still validate from headers

Constant-size state accumulator

The entire coin set is committed in an O(1) cryptographic accumulator, so validating state never grows with history.

  • RSA-2048 group of unknown order — add-only, succinct membership
  • Header commitments — AccRoot, NullRoot (spent set), CMRoot (ZK tree), PQAccRoot, PoRRoot
  • Class-group target — trustless, no-setup group (production roadmap)

Reorg safety & conservation

Chain reorganizations roll back and replay state atomically, and every block proves no coins were created from nothing.

  • Snapshot + rollback — all state (UTXO, nullifiers, ZK tree, PQ set) reorg-restored
  • Bounded reorg depth — PoW seed lag ≥ max reorg keeps seeds branch-stable
  • Coinbase conservation proof — supply invariant: new coins = base reward only

3 · Privacy & Zero-Knowledge

the endgame: hide who, what, and the link

Transparent zk-STARK engine

A from-scratch proof system with no trusted setup and post-quantum security — only hashing and Reed-Solomon, nothing breakable by a quantum computer.

  • Goldilocks field + NTT, FRI low-degree test, DEEP-ALI AIR engine
  • Poseidon-over-Goldilocks hash (width-3 & width-8)
  • Zero-knowledge masking — coset LDE + Z_H·r so proofs leak nothing about the witness
  • FRI grinding — ~112-bit conjectured soundness

Anonymous commitment-tree spend

Spend a coin by proving — in zero knowledge — that it's somewhere in the tree, revealing only a nullifier. No ring, no decoys: the anonymity set is the whole chain.

  • Poseidon commitment tree with 256-bit nodes (2¹²⁸ collision-resistant)
  • Epoch sharding — fixed-depth trees roll over → unlimited coins at constant proof cost
  • Nullifier set — double-spend protection without revealing which coin

Confidential amounts HIDDEN VALUE

Transaction amounts are hidden end-to-end, proven conserved entirely in-field (no fragile cross-system binding).

  • In-field value balance — Σin = Σout + fee over hidden amounts
  • Range proofs — bit-decomposition rules out inflation / field-wrap
  • Fused spend — one STARK spends a hidden-value coin and mints a hidden-value coin

Recipient-secret nullifier UNLINKABLE

Removes sender↔spend linkability: the nullifier derives from a recipient secret the sender never learns, so the sender can't detect when their payment is later spent.

  • Two-key note (Zcash-style, all-Poseidon, no ECC): address pk = H(nk)
  • nf = H(nk, ρ) — needs the recipient's secret nk
  • Spend authority — only the nk-holder can spend (a thief who knows the address cannot)
  • Merged circuit — confidential and unlinkable in one proof

Stealth addresses

Every payment lands at a fresh one-time key, so an address is never reused on-chain and recipients are unlinkable.

  • One-time keys from a sender↔recipient shared secret
  • View tags — fast scan hint
  • Encrypted amount — recipient recovers a hidden value off-chain

Ring & Pedersen path (classical)

A parallel sound transparent path for spends, retained alongside the ZK endgame.

  • Triptych-style one-out-of-many ring spend (sender-anonymous)
  • Pedersen confidential amounts + Schnorr conservation
  • Shared key-image set — a coin can't be spent both ways

4 · Economics & Incentives

emission, yield & alignment

Emission & supply

A smooth, decreasing reward with a perpetual tail — predictable issuance with a hard cap before tail emission.

  • Smooth decay reward = remainingSupply ≫ shift
  • Supply cap ~18.4M OBX + perpetual tail floor for security funding
  • Overflow-checked coinbase economics

Confidential staking vaults PRIVATE YIELD

A built-in DeFi primitive: lock OBX to earn yield from the incentive pool — the staked amount and yield stay private.

  • Time-locked deposits, claimable principal + yield at maturity
  • Yield from the incentive pool — bounded so it can never inflate supply
  • Proceeds re-enter as fresh stealth outputs

Incentive pool & referrals

A slice of each block funds long-term holders and an optional, sybil-guarded viral bonus.

  • Holding bonus — lock-duration-weighted, funded from 5% of reward
  • Referral bonus — capped + decaying, off by default until sybil-bound
  • Fees recycled to miners; supply invariant preserved

7 · Storage & Scaling

stay light without losing trustlessness

3-tier pruning

Nodes can run light: old data is shed in tiers while the cryptographic commitments keep verification trustless.

  • Tier 0 — evict cold blocks from RAM
  • Tier 1 — Merkle frontier + disk-backed sets
  • Tier 2 — snapshot + block-body pruning

Snapshot sync & constant state

New nodes sync to a committed snapshot instead of replaying all history; state size is independent of coin count.

  • State snapshots committed + restorable on reorg
  • Constant-size accumulator — node state O(depth), not O(coins)
  • Bolt persistence, durable write-ahead ordering

Epoch-sharded anonymity

The commitment tree shards into fixed-depth epochs, giving unlimited total coins at a constant per-proof cost.

  • Roll-on-full trees — proof depth never grows
  • All epoch roots are anchors; finalized roots permanent
